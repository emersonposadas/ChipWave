Dónde se procesa el NSF

GitHub Actions solo descarga archivos NSF o compila el emulador. El análisis ocurre en el navegador: la app hace fetch() del NSF, lee los bytes, parsea el header, pasa el archivo a Game_Music_Emu y renderiza audio con Web Audio API.

Si gme_mute_voice está disponible en vendor/libgme.js, los botones usan mute real. Si no está disponible, la app usa fallback aproximado por bandas.
