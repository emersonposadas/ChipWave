# ChipWave NSF Visualizer

Complete current source files for the GitHub Pages NSF visualizer.

## Included

```text
index.html
styles.css
app.js
.nojekyll
nsf/catalog.json
vendor/README.md
.github/workflows/build-libgme.yml
.github/workflows/import-nsf.yml
```

## Not included

```text
vendor/libgme.js
vendor/libgme.wasm
```

Those are generated by the successful GitHub Actions workflow in your repo.

## Required final repo structure

```text
/
├── index.html
├── styles.css
├── app.js
├── .nojekyll
├── vendor/
│   ├── libgme.js      # generated by workflow
│   └── libgme.wasm    # generated by workflow
├── nsf/
│   ├── catalog.json
│   └── *.nsf          # imported by workflow
└── .github/
    └── workflows/
        ├── build-libgme.yml
        └── import-nsf.yml
```

## Steps

1. Copy these files into your repo.
2. Keep the generated `vendor/libgme.js` and `vendor/libgme.wasm` from your successful workflow run.
3. Commit and push.
4. Open GitHub Pages.
5. Import NSF files using `Import NSF`.
6. Reload the catalog in the app.

## Real mute

The app tries to use:

```text
gme_mute_voice
```

If available in `vendor/libgme.js`, mute buttons use real GME voice mute.

If not available, the app falls back to approximate band mute.
